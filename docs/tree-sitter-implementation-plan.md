## Overview

Build a new `PredictableTreeSitter` library in the predictable-code workspace that:

1. Wraps tree-sitter's C API via Lean 4 FFI
2. Bundles Java, Python, and Kotlin grammar parsers as compiled C code
3. Defines typed grammar schemas and a `GrammarSpec` typeclass with proof obligations
4. Extracts declarations from source code with source-location tracking
5. Provides source maps between host language and generated Lean 4 verification code
6. Proves mapping totality, cross-language consistency, and source map properties

### Dependency Chain (after implementation)

```
PredictableCore ─┬─→ PredictableAgents ─┬─→ PredictableCli → Main
                 │                      │
PredictableTUI ──┴──────────────────────┘

PredictableVerification (standalone)

PredictableTreeSitter (standalone — no project deps, only C FFI)
   └── ffi/ (tree-sitter core + Java/Python/Kotlin grammars compiled to C)
```

`PredictableTreeSitter` is standalone initially. Integration with `PredictableVerification` and `PredictableCli` comes in future phases.

### Vendoring Strategy (inspired by [tree-sitter-language-pack](https://github.com/Goldziher/tree-sitter-language-pack))

Instead of git submodules, we use a **vendored source approach**:

1. `language_definitions.json` declares each grammar's repo URL, pinned commit hash, and optional subdirectory
2. `vendor_grammars.sh` clones each repo, checks out the pinned revision, copies only `src/` (parser.c, scanner.c), and discards the rest
3. Vendored `parsers/` directory is committed to the repo (no submodule init needed)
4. Tree-sitter core `lib/` is also vendored (just the `include/` and `src/` directories)

**Advantages over git submodules:**
- No `git submodule init --recursive` in CI — cloning the repo is sufficient
- Smaller checkout (only the C source files we need, not full grammar repos)
- Adding a new language = one JSON entry + re-run vendor script
- Pinned revisions give reproducible builds without submodule tracking issues
- No nested `.git` directories or submodule sync headaches

### C Build Pipeline

```
ffi/tree-sitter/lib/src/lib.c        ──→  ts_core.o
ffi/parsers/java/src/parser.c        ──→  ts_java.o
ffi/parsers/python/src/parser.c      ──→  ts_python.o
ffi/parsers/python/src/scanner.c     ──→  ts_python_scanner.o
ffi/parsers/kotlin/src/parser.c      ──→  ts_kotlin.o
ffi/parsers/kotlin/src/scanner.c     ──→  ts_kotlin_scanner.o
ffi/shim.c                           ──→  ts_shim.o

All .o files ──→  ar rcs ──→  libtree-sitter-lean.a  (static library)
```

Lake links `libtree-sitter-lean.a` into the `PredictableTreeSitter` lean_lib.

## Phase 0 — C Infrastructure & Build System

**Goal:** Get tree-sitter core + three grammars compiling as a static C library, and update Lake/Makefile to build it.

### Tasks

#### 0.1 Create Grammar Registry (`ffi/language_definitions.json`)

A JSON file declaring each grammar's source repository, pinned revision, and build metadata.
Modeled after [tree-sitter-language-pack's `language_definitions.json`](https://github.com/Goldziher/tree-sitter-language-pack/blob/main/sources/language_definitions.json):

```json
{
  "tree-sitter-core": {
    "repo": "https://github.com/tree-sitter/tree-sitter",
    "rev": "<pinned-commit-hash>",
    "paths": ["lib/include", "lib/src"]
  },
  "java": {
    "repo": "https://github.com/tree-sitter/tree-sitter-java",
    "rev": "e10607b45ff745f5f876bfa3e94fbcc6b44bdc11",
    "has_scanner": false
  },
  "python": {
    "repo": "https://github.com/tree-sitter/tree-sitter-python",
    "rev": "26855eabccb19c6abf499fbc5b8dc7cc9ab8bc64",
    "has_scanner": true
  },
  "kotlin": {
    "repo": "https://github.com/fwcd/tree-sitter-kotlin",
    "rev": "57fb4560ba8641865bc0baa6b3f413b236112c4c",
    "has_scanner": true
  }
}
```

Revision hashes taken from tree-sitter-language-pack's pinned versions (known-good, ABI-compatible).

#### 0.2 Create Vendor Script (`ffi/vendor_grammars.sh`)

A shell script (no Python/npm dependency) that:

1. Reads `language_definitions.json`
2. For each grammar: shallow-clones the repo, checks out the pinned revision, copies `src/` to `parsers/<lang>/src/`
3. For tree-sitter core: copies `lib/include/` and `lib/src/` to `ffi/tree-sitter/lib/`
4. Fixes any relative `#include` paths (e.g., `../../common/` → `common/`)
5. Cleans up temporary clone directories

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
DEFINITIONS="$SCRIPT_DIR/language_definitions.json"
PARSERS_DIR="$SCRIPT_DIR/parsers"
VENDOR_TMP="$SCRIPT_DIR/.vendor-tmp"

# Clean previous state
rm -rf "$VENDOR_TMP"
mkdir -p "$VENDOR_TMP" "$PARSERS_DIR"

# Clone and extract each grammar using jq to parse JSON
for lang in $(jq -r 'keys[] | select(. != "tree-sitter-core")' "$DEFINITIONS"); do
  repo=$(jq -r ".\"$lang\".repo" "$DEFINITIONS")
  rev=$(jq -r ".\"$lang\".rev" "$DEFINITIONS")
  dir=$(jq -r ".\"$lang\".directory // empty" "$DEFINITIONS")

  echo "Vendoring $lang from $repo@$rev"
  git clone --depth 1 "$repo" "$VENDOR_TMP/$lang"
  (cd "$VENDOR_TMP/$lang" && git fetch --depth 1 origin "$rev" && git checkout "$rev")

  src_dir="$VENDOR_TMP/$lang/${dir:+$dir/}src"
  mkdir -p "$PARSERS_DIR/$lang/src"
  cp "$src_dir"/parser.c "$PARSERS_DIR/$lang/src/"
  [ -f "$src_dir/scanner.c" ] && cp "$src_dir/scanner.c" "$PARSERS_DIR/$lang/src/"
  [ -d "$src_dir/tree_sitter" ] && cp -r "$src_dir/tree_sitter" "$PARSERS_DIR/$lang/src/"
done

# Vendor tree-sitter core
# ... (similar pattern for core library)

rm -rf "$VENDOR_TMP"
echo "Done. Vendored parsers in $PARSERS_DIR"
```

The vendored `parsers/` and `tree-sitter/` directories are committed to git.
To update grammars: change the `rev` in `language_definitions.json` and re-run the script.

A Makefile target wraps this:

```makefile
vendor-grammars:
	cd packages/ffi && bash vendor_grammars.sh
```

#### 0.2 Write the C Shim (`ffi/shim.c`)

The shim bridges Lean 4's runtime types to tree-sitter's C API. It handles:

- **External class registration** with finalizers for `TSParser*`, `TSTree*`, `TSQuery*`, `TSQueryCursor*`
- **String conversion** between Lean strings (`lean_string_cstr`) and C strings
- **TSNode value passing** — `TSNode` is a 24-byte struct (not a pointer), so the shim must box/unbox it into a Lean external object or pass fields individually
- **Error handling** — return `lean_io_result_mk_error` on failures

Key shim functions:

```c
#include <lean/lean.h>
#include <tree_sitter/api.h>

// --- External class registration with finalizers ---

static void ts_parser_finalizer(void *ptr) { ts_parser_delete((TSParser *)ptr); }
static void ts_tree_finalizer(void *ptr) { ts_tree_delete((TSTree *)ptr); }
static void ts_query_finalizer(void *ptr) { ts_query_delete((TSQuery *)ptr); }
static void ts_query_cursor_finalizer(void *ptr) { ts_query_cursor_delete((TSQueryCursor *)ptr); }
static void noop_foreach(void *data, b_lean_obj_arg arg) { /* no-op */ }

// One external class per type, lazily initialized
static lean_external_class *g_ts_parser_class = NULL;
static lean_external_class *g_ts_tree_class = NULL;
// ... etc.

// --- Parser functions ---

LEAN_EXPORT lean_obj_res lean_ts_parser_new(lean_obj_arg world) {
    TSParser *p = ts_parser_new();
    if (!p) return lean_io_result_mk_error(lean_mk_io_user_error(lean_mk_string("ts_parser_new failed")));
    if (!g_ts_parser_class)
        g_ts_parser_class = lean_register_external_class(&ts_parser_finalizer, &noop_foreach);
    return lean_io_result_mk_ok(lean_alloc_external(g_ts_parser_class, p));
}

LEAN_EXPORT lean_obj_res lean_ts_parser_set_language(b_lean_obj_arg parser, b_lean_obj_arg lang, lean_obj_arg world) {
    TSParser *p = (TSParser *)lean_get_external_data(parser);
    const TSLanguage *l = (const TSLanguage *)lean_get_external_data(lang);
    bool ok = ts_parser_set_language(p, l);
    return lean_io_result_mk_ok(lean_box(ok ? 1 : 0));
}

LEAN_EXPORT lean_obj_res lean_ts_parser_parse_string(
    b_lean_obj_arg parser, b_lean_obj_arg source, lean_obj_arg world) {
    TSParser *p = (TSParser *)lean_get_external_data(parser);
    const char *src = lean_string_cstr(source);
    size_t len = lean_string_size(source) - 1; // exclude null terminator
    TSTree *tree = ts_parser_parse_string(p, NULL, src, (uint32_t)len);
    if (!tree) return lean_io_result_mk_error(lean_mk_io_user_error(lean_mk_string("parse failed")));
    if (!g_ts_tree_class)
        g_ts_tree_class = lean_register_external_class(&ts_tree_finalizer, &noop_foreach);
    return lean_io_result_mk_ok(lean_alloc_external(g_ts_tree_class, tree));
}

// --- Language accessors ---

extern const TSLanguage *tree_sitter_java(void);
extern const TSLanguage *tree_sitter_python(void);
extern const TSLanguage *tree_sitter_kotlin(void);

// Wrap each language pointer in a "fake" external object (no finalizer — languages are static)
static lean_external_class *g_ts_language_class = NULL;
static void noop_finalizer(void *ptr) { /* languages are static, never free */ }

LEAN_EXPORT lean_obj_res lean_tree_sitter_java(lean_obj_arg world) {
    if (!g_ts_language_class)
        g_ts_language_class = lean_register_external_class(&noop_finalizer, &noop_foreach);
    return lean_io_result_mk_ok(
        lean_alloc_external(g_ts_language_class, (void *)tree_sitter_java()));
}

LEAN_EXPORT lean_obj_res lean_tree_sitter_python(lean_obj_arg world) {
    if (!g_ts_language_class)
        g_ts_language_class = lean_register_external_class(&noop_finalizer, &noop_foreach);
    return lean_io_result_mk_ok(
        lean_alloc_external(g_ts_language_class, (void *)tree_sitter_python()));
}

LEAN_EXPORT lean_obj_res lean_tree_sitter_kotlin(lean_obj_arg world) {
    if (!g_ts_language_class)
        g_ts_language_class = lean_register_external_class(&noop_finalizer, &noop_foreach);
    return lean_io_result_mk_ok(
        lean_alloc_external(g_ts_language_class, (void *)tree_sitter_kotlin()));
}

// --- Node functions ---
// TSNode is a struct (not pointer), so we box it into a lean external object
// containing a heap-allocated copy of the TSNode struct.

typedef struct { TSNode node; } BoxedTSNode;

static lean_external_class *g_ts_node_class = NULL;
static void ts_node_finalizer(void *ptr) { free(ptr); }

static lean_obj_res box_ts_node(TSNode node) {
    if (!g_ts_node_class)
        g_ts_node_class = lean_register_external_class(&ts_node_finalizer, &noop_foreach);
    BoxedTSNode *boxed = (BoxedTSNode *)malloc(sizeof(BoxedTSNode));
    boxed->node = node;
    return lean_alloc_external(g_ts_node_class, boxed);
}

static TSNode unbox_ts_node(b_lean_obj_arg obj) {
    return ((BoxedTSNode *)lean_get_external_data(obj))->node;
}

LEAN_EXPORT lean_obj_res lean_ts_tree_root_node(b_lean_obj_arg tree, lean_obj_arg world) {
    TSTree *t = (TSTree *)lean_get_external_data(tree);
    TSNode root = ts_tree_root_node(t);
    return lean_io_result_mk_ok(box_ts_node(root));
}

LEAN_EXPORT lean_obj_res lean_ts_node_type(b_lean_obj_arg node, lean_obj_arg world) {
    TSNode n = unbox_ts_node(node);
    const char *type_str = ts_node_type(n);
    return lean_io_result_mk_ok(lean_mk_string(type_str));
}

LEAN_EXPORT lean_obj_res lean_ts_node_child_count(b_lean_obj_arg node, lean_obj_arg world) {
    TSNode n = unbox_ts_node(node);
    uint32_t count = ts_node_child_count(n);

}

LEAN_EXPORT lean_obj_res lean_ts_node_named_child_count(b_lean_obj_arg node, lean_obj_arg world) {
    TSNode n = unbox_ts_node(node);
    uint32_t count = ts_node_named_child_count(n);

}

    TSNode n = unbox_ts_node(node);
    TSNode child = ts_node_child(n, index);

LEAN_EXPORT lean_obj_res lean_ts_node_named_child(b_lean_obj_arg node, uint32_t index, lean_obj_arg world) {

// --- Query functions ---

LEAN_EXPORT lean_obj_res lean_ts_query_new(
    b_lean_obj_arg lang, b_lean_obj_arg source, lean_obj_arg world) {
    const TSLanguage *l = (const TSLanguage *)lean_get_external_data(lang);
    const char *src = lean_string_cstr(source);
    uint32_t error_offset;

        return lean_io_result_mk_error(lean_mk_io_user_error(lean_mk_string(buf)));

        g_ts_query_class = lean_register_external_class(&ts_query_finalizer, &noop_foreach);
```

## Phase 4 — Extraction Engine

**Goal:** A generic function that takes source code + language, parses with tree-sitter, walks the CST, and returns `Array Declaration` with source locations.

### Tasks

#### 4.1 `Extract/Extract.lean`

```lean
namespace PredictableTreeSitter.Extract

def parseSource
    (lang : IO FFI.TSLanguage)
    (source : String)
    : IO FFI.TSTree := do
  let parser ← FFI.TSParser.new
  let language ← lang
  let ok ← parser.setLanguage language
  if !ok then throw (IO.userError "language version mismatch")
  parser.parseString source

-- Walk the tree recursively, collecting declarations
-- Uses the GrammarSpec Node type to map node-type strings to DeclarationType
partial def walkNode
    (node : FFI.TSNode)
    (nodeMapping : String → Option DeclarationType)
    (source : String)
    : IO (Array Declaration) := do
  let isNull ← node.isNull
  if isNull then return #[]
  let nodeType ← node.type
  let childCount ← node.childCount
  let mut results : Array Declaration := #[]
  -- Check if this node is a declaration
  match nodeMapping nodeType with
  | some declType =>
    let startRow ← node.startRow
    let startCol ← node.startColumn
    let endRow ← node.endRow
    let endCol ← node.endColumn
    let name ← extractName node  -- find identifier child
    let children ← collectChildDeclarations node nodeMapping source
    results := results.push {
      name := name
      declType := declType
      source := {
        startLine := startRow.toNat + 1
        startColumn := startCol.toNat
        endLine := endRow.toNat + 1
        endColumn := endCol.toNat
      }
      children := children
    }
  | none =>
    -- Not a declaration; recurse into children
    for i in List.range childCount.toNat do
      let child ← node.child i.toUInt32
      let childResults ← walkNode child nodeMapping source
      results := results ++ childResults
  return results

-- Build the string→DeclarationType mapping from a GrammarSpec instance
-- This maps tree-sitter node-type strings to our typed Node values
def nodeStringMapping (Node : Type) [GrammarSpec Node] [Repr Node]
    : String → Option DeclarationType :=
  -- Implementation: for each node in declarationNodes, map its repr string
  -- to its DeclarationType
  fun s => ...  -- match on known strings

-- Top-level extraction entry point
def extractDeclarations
    (lang : IO FFI.TSLanguage)
    (nodeMapping : String → Option DeclarationType)
    (source : String)
    : IO (Array Declaration) := do
  let tree ← parseSource lang source
  let root ← tree.rootNode
  walkNode root nodeMapping source

-- Convenience functions per language
def extractJava (source : String) : IO (Array Declaration) :=
  extractDeclarations treeSitterJava javaNodeMapping source

def extractPython (source : String) : IO (Array Declaration) :=
  extractDeclarations treeSitterPython pythonNodeMapping source

def extractKotlin (source : String) : IO (Array Declaration) :=
  extractDeclarations treeSitterKotlin kotlinNodeMapping source

end PredictableTreeSitter.Extract
```

```lean
namespace PredictableTreeSitter.SourceMap

def SourceMap.lookupByLean (sm : SourceMap) (loc : LeanLocation) : Option SourceMapping :=
  sm.entries.find? fun m => m.lean == loc

def SourceMap.lookupBySource (sm : SourceMap) (loc : SourceLocation) : Option SourceMapping :=
  sm.entries.find? fun m => m.source == loc

def SourceMap.traceError (sm : SourceMap) (leanLine : Nat) : Option SourceLocation :=
  match sm.entries.find? fun m => m.lean.line == leanLine with
  | some mapping => some mapping.source
  | none =>
    -- Find nearest preceding entry
    let preceding := sm.entries.filter fun m => m.lean.line ≤ leanLine
    preceding.back?.map (·.source)
```

```lean
def SourceMap.compose (sm1 sm2 : SourceMap) : SourceMap :=
  -- For each entry in sm2, look up the corresponding entry in sm1
  -- to create a transitive mapping: source → intermediate → lean
  { entries := sm2.entries.filterMap fun e2 =>
      match sm1.lookupBySource e2.source with  -- simplified; real impl matches on lean location
      | some e1 => some { source := e1.source, lean := e2.lean,
```

## Reference: Lean 4 FFI Patterns

### Opaque Type Declaration

```lean
opaque FooPointed : NonemptyType
def Foo := FooPointed.type
instance : Nonempty Foo := FooPointed.property
```

### Extern Function Binding

```lean
@[extern "c_function_name"]
opaque leanFunctionName (arg1 : @& Type1) (arg2 : Type2) : IO ReturnType
```

- `@&` marks **borrowed** parameters (no reference count change → `b_lean_obj_arg` in C)
- Without `@&`, parameter is **consumed** (reference count transferred → `lean_obj_arg` in C)
- `IO` return type passes a `lean_obj_arg world` parameter and expects `lean_io_result_mk_ok/error`

### C Shim Patterns

```c
#include <lean/lean.h>

// Register external class (once, lazily)
static lean_external_class *g_foo_class = NULL;
static void foo_finalizer(void *ptr) { free_foo((Foo *)ptr); }
static void noop_foreach(void *data, b_lean_obj_arg arg) {}

// Wrap C pointer as Lean object
lean_obj_res wrap_foo(Foo *ptr) {
    if (!g_foo_class)
        g_foo_class = lean_register_external_class(&foo_finalizer, &noop_foreach);
    return lean_alloc_external(g_foo_class, ptr);
}

// Unwrap Lean object to C pointer
Foo *unwrap_foo(b_lean_obj_arg obj) {
    return (Foo *)lean_get_external_data(obj);
}

// IO function: takes world, returns lean_io_result_mk_ok/error
LEAN_EXPORT lean_obj_res lean_create_foo(lean_obj_arg world) {
    Foo *f = create_foo();
    if (!f) return lean_io_result_mk_error(lean_mk_io_user_error(lean_mk_string("failed")));
    return lean_io_result_mk_ok(wrap_foo(f));
}
```

### Lake C Compilation

```lean
-- Compile a .c file to .o
target myfile.o (pkg : NPackage _package.name) : FilePath := do
  let src := pkg.dir / "ffi" / "myfile.c"
  let obj := pkg.buildDir / "ffi" / "myfile.o"
  buildFileAfterDep obj (<- inputFile src) fun src => do
    let leanDir := (<- getLeanIncludeDir).toString
    compileO "description" obj src #["-I", leanDir, "-fPIC", "-O2"]

-- Bundle .o files into a static library
extern_lib mylib (pkg : NPackage _package.name) := do
  let name := nameToStaticLib "mylib"
  let obj1 ← fetch <| pkg.target ``myfile.o
  buildStaticLib (pkg.buildDir / "lib" / name) #[obj1]

-- Lean library that depends on the C library
lean_lib MyLeanLib where
  extraDepTargets := #[`mylib]
```
